# SPEAKAI — Development Improvement Plan

> هذا الملف مخصص للمشاركة مع أداة برمجة وصفية (vibe-coding agent) لتنفيذ التحسينات تدريجياً. كل بند مكتوب كمهمة مستقلة قابلة للتنفيذ.

---

## P0 — إصلاحات عاجلة (Critical Fixes)

### 1. تصحيح قائمة نماذج Gemini في `/app/api/evaluate/route.ts`
- **المشكلة:** القائمة `modelsToTry` تحتوي `gemini-3.5-flash` وهو غير موجود فعلياً، مما يسبب فشل/تأخير دائم في أول محاولة استدعاء.
- **المطلوب:** تحديث القائمة إلى نماذج Gemini الفعلية الحالية المتاحة في حساب المشروع (تحقق من `@google/genai` docs)، بترتيب من الأسرع إلى الأدق.
- **معيار القبول:** أول استدعاء API ينجح من المحاولة الأولى بدون اللجوء لـ fallback.

### 2. تحسين معالجة الأخطاء في `submitForEvaluation`
- **المطلوب:** تمييز رسائل الخطأ حسب النوع (لا اتصال إنترنت / رفض الميكروفون / فشل الـ API / استجابة غير صالحة) بدل رسالة عامة واحدة.
- **معيار القبول:** كل نوع خطأ يعرض رسالة مختلفة وواضحة للمستخدم بلغته (عربي/إنكليزي).

---

## P1 — توسعة المحتوى (Content Expansion)

### 3. إعادة بناء `lib/sentences.ts` كنظام مصنّف
- **المطلوب:**
  - تحويل البنية من array بسيط إلى نظام فئات: `category` (داily-conversation, business, travel, news, movies) و `difficulty` (beginner, intermediate, advanced).
  - إضافة 100-150 جملة موزعة على الفئات والمستويات.
  - دعم اختيار المستخدم لفئة/مستوى قبل بدء الجلسة.
- **معيار القبول:** المستخدم يقدر يفلتر الجمل حسب الفئة والمستوى من واجهة `/practice`.

### 4. توليد محتوى ديناميكي عبر Gemini
- **المطلوب:** إنشاء API route جديد (`/api/generate-content`) يولّد جمل/فقرات جديدة بناءً على مستوى المستخدم وتاريخ أخطائه السابقة (من IndexedDB).
- **معيار القبول:** يقدر المستخدم يطلب "جمل جديدة" وتظهر جمل غير مكررة من البنك الثابت.

### 5. دعم فقرات متصلة (Multi-sentence shadowing)
- **المطلوب:** توسعة نموذج `Sentence` لدعم فقرات من 2-4 جمل متتالية كوحدة شادوينغ واحدة، مع تسجيل وتقييم متواصل.
- **معيار القبول:** يوجد نوع جلسة جديد "Paragraph Mode" بجانب "Sentence Mode" الحالي.

---

## P2 — جودة الصوت (Audio Quality)

### 6. استبدال `speechSynthesis` بـ Neural TTS
- **المشكلة:** الصوت الحالي يعتمد على TTS المتصفح/النظام، غير ثابت الجودة وغير طبيعي.
- **المطلوب:**
  - إنشاء API route (`/api/tts`) يستخدم Gemini TTS أو خدمة TTS عصبية (ElevenLabs/Google Cloud Neural2) لتوليد صوت طبيعي للجملة.
  - الإبقاء على `speechSynthesis` كـ fallback عند فشل الـ API أو في حال عدم توفر مفتاح API.
  - دعم الكاش (cache) للجمل المتكررة لتفادي استدعاءات API مكررة لنفس النص.
- **معيار القبول:** صوت الجملة المسموع طبيعي وليس آلياً، مع نفس عناصر التحكم بالسرعة الحالية.

### 7. مقارنة صوت المستخدم بالأصلي (A/B playback)
- **المطلوب:** في شاشة الـ Feedback، إضافة زرين "🔊 الأصلي" و "🎤 صوتك" لتشغيل الصوتين بالتتابع للمقارنة المباشرة.
- **معيار القبول:** المستخدم يقدر يسمع صوته وصوت المرجع بضغطة واحدة لكل واحد.

---

## P3 — تقييم متقدم (Advanced Evaluation)

### 8. تحليل على مستوى الكلمة
- **المطلوب:** تعديل الـ system instruction في `/api/evaluate/route.ts` ليرجع schema موسّع يتضمن مصفوفة `words` فيها لكل كلمة: `{ word, status: correct|mispronounced|missing }`.
- **المطلوب في الواجهة:** عرض الجملة بالألوان (أخضر/أصفر/أحمر) حسب نتيجة كل كلمة في شاشة الـ Feedback.
- **معيار القبول:** عند ظهور النتيجة، الجملة الأصلية تظهر بألوان لكل كلمة حسب دقة النطق.

### 9. تتبع نقاط الضعف الصوتية (Phoneme Weakness Tracking)
- **المطلوب:**
  - توسعة `lib/db.ts` بجدول جديد `phonemeIssues` لتسجيل الأصوات المتكررة الخاطئة.
  - إضافة قسم في صفحة `/result` يعرض "أكثر 3 أصوات تحتاج تحسين" بناءً على تاريخ المستخدم.
- **معيار القبول:** بعد عدة جلسات، تظهر للمستخدم نمط واضح لأخطائه المتكررة.

---

## P4 — البنية التقنية والموثوقية (Technical Infrastructure)

### 10. اختبارات وحدة أساسية
- **المطلوب:** إضافة اختبارات لـ `lib/streak.ts` (checkStreakValidity, recordSessionCompletion) و لمنطق تحليل JSON في `route.ts`.
- **معيار القبول:** `npm test` يعمل ويغطي أهم منطق الأعمال.

### 11. ضغط الصوت قبل الرفع
- **المطلوب:** تقليل حجم ملف الصوت (bitrate/sample rate) قبل إرساله لـ `/api/evaluate` لتسريع الرفع وخفض تكلفة استدعاء API.
- **معيار القبول:** حجم الملف المرسل أصغر بشكل ملموس بدون فقدان جودة كافية للتحليل.

### 12. حسابات ومزامنة سحابية (اختياري)
- **المطلوب:** دمج خدمة auth خفيفة (Supabase/Firebase) لمزامنة `attempts` و `streak` عبر الأجهزة، مع إبقاء الوضع المحلي (IndexedDB) كخيار افتراضي بدون حساب.
- **معيار القبول:** تسجيل الدخول اختياري، والبيانات المحلية تُرفع للسحابة عند تسجيل الدخول لأول مرة.

---

## P5 — تحفيز وتجربة المستخدم (Engagement)

### 13. نظام إنجازات (Achievements)
- **المطلوب:** إضافة جدول `achievements` في `lib/db.ts` ومنطق فتح شارات (مثل: 7 أيام متواصلة، 50 محاولة، متوسط درجات 90+).
- **معيار القبول:** ظهور إشعار/شارة عند تحقيق شرط معين.

### 14. مشاركة نتيجة الجلسة
- **المطلوب:** زر "مشاركة" في صفحة `/result` يولّد صورة/بطاقة ملخصة (Canvas) قابلة للحفظ أو المشاركة.
- **معيار القبول:** يقدر المستخدم يحفظ صورة بنتيجته من نفس الصفحة.
